# Data Model: Characters Catalog

**Branch**: `013-characters-catalog` | **Date**: 2026-06-24

One new table (`characters`) + one partial index. R2 path contract for the
photos. No changes to existing tables in this module — the FK in
`twists` is added by the **module 005 delta**, not here.

---

## Table — `characters`

```sql
CREATE TABLE characters (
    id           INTEGER     PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    slug         TEXT        NOT NULL UNIQUE
                 CHECK (slug ~ '^[a-z0-9-]{2,40}$'),
    display_name TEXT        NOT NULL
                 CHECK (char_length(display_name) BETWEEN 2 AND 60),
    photo_r2_key TEXT        NOT NULL
                 CHECK (photo_r2_key LIKE 'static/characters/%.webp'),
    aspect_ratio TEXT        NOT NULL DEFAULT '1:1'
                 CHECK (aspect_ratio IN ('1:1', '9:16', '16:9')),
    active       BOOLEAN     NOT NULL DEFAULT TRUE,
    sort_order   INTEGER     NOT NULL DEFAULT 100,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_characters_active_sort
    ON characters (sort_order, id)
    WHERE active = TRUE;
```

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | INTEGER | PK, IDENTITY | Stable integer key. Used by FK in `twists` (module 005 delta). |
| `slug` | TEXT | UNIQUE, regex `^[a-z0-9-]{2,40}$` | Used as filename (`<slug>.webp`) and in R2 path. **MUST NOT change once shipped** — see plan R-3. |
| `display_name` | TEXT | length 2-60 | Rendered in the carousel and intro overlay. UTF-8 (accents, ñ, etc.). |
| `photo_r2_key` | TEXT | NOT NULL, starts with `static/characters/` and ends with `.webp` | Joined with `R2_PUBLIC_BASE_URL` at serialization time to form `photo_url`. |
| `aspect_ratio` | TEXT | one of `1:1 / 9:16 / 16:9`, default `1:1` | Persisted so a future per-character override (cinematic landscape) is a single UPDATE, not a schema change. MVP always uses `1:1`. |
| `active` | BOOLEAN | NOT NULL, default TRUE | Soft-delete flag (Gate 7). Filtered at the SQL level via the partial index. |
| `sort_order` | INTEGER | NOT NULL, default 100 | Lower = first in the carousel. Default 100 gives manual ops room to insert at front (sort=50) or end (sort=150). |
| `created_at` | TIMESTAMPTZ | NOT NULL, default NOW() | Audit. |
| `updated_at` | TIMESTAMPTZ | NOT NULL, default NOW() | Updated by trigger on `active` / `sort_order` changes (see below). |

### Trigger — bump `updated_at`

```sql
CREATE TRIGGER characters_set_updated_at
BEFORE UPDATE OF active, sort_order, display_name, photo_r2_key, aspect_ratio
ON characters
FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```

`set_updated_at()` is defined globally in module 001's migration (already in
use by `users`, `chapters`, etc.). No new function is created here.

### Index rationale

The single query the endpoint runs is:

```sql
SELECT id, slug, display_name, photo_r2_key, aspect_ratio
FROM characters
WHERE active = TRUE
ORDER BY sort_order ASC, id ASC;
```

The partial index `idx_characters_active_sort` is index-only-scannable for
this query: PG covers `(sort_order, id)` from the index and falls back to
the heap for the remaining 3 columns. With ≤12 active rows this is
effectively a sequential scan; the index is there for **correctness +
clarity**, not performance.

## R2 path contract — character photos

```
static/characters/{slug}.webp
```

| Segment | Source | Constraints |
|---|---|---|
| `static/` | Constant prefix | Shared with `placeholder.mp4`, `placeholder.webp`, `outro.mp4`, `intro_bg.png` |
| `characters/` | Constant sub-prefix | Segregates the catalog from other static assets |
| `{slug}` | `characters.slug` | kebab-case, must match `^[a-z0-9-]{2,40}$` |
| `.webp` | Fixed extension | WebP only in MVP; PNG/JPEG are rejected by the script (FR-004) |

**Example**:
```
static/characters/messi.webp
```

**Invariants**:

1. **Deterministic**: `slug` is the primary key of the path. The mapping is
   `slug → filesystem name → R2 key`. Renaming `slug` would orphan the R2
   object — forbidden by R-3 in plan.
2. **Immutable per slug**: once `static/characters/messi.webp` is uploaded,
   future migrations may reference it without re-uploading. Replacement
   (e.g., better photo) happens by uploading new bytes to the same key;
   Cloudflare CDN cache is purged manually (rare op, accepted cost).
3. **Public read**: served via the same public R2 bucket as the rest of
   `static/*`. No signed URLs needed.

## Backward / forward compatibility

- **Existing data**: tables in main (e.g., `twists`) are not
  touched here. The FK that depends on `characters.id` is added by
  module 005 delta in a separate migration.
- **Future v0.2 admin UI**: when admin endpoints land, the table is
  ready (`updated_at`, `sort_order`, `active`). No schema change needed.
- **Future per-season characters**: when/if MVP graduates from "global
  catalog" to "per-season cast", a join table
  `season_characters(season_id, character_id, ...)` is added without
  touching `characters` itself.

## Sample seed row (illustrative — full seed in tasks.md T-002)

```sql
INSERT INTO characters (slug, display_name, photo_r2_key, sort_order)
VALUES ('messi', 'Lionel Messi', 'static/characters/messi.webp', 10)
ON CONFLICT (slug) DO UPDATE SET
    display_name = EXCLUDED.display_name,
    photo_r2_key = EXCLUDED.photo_r2_key,
    sort_order   = EXCLUDED.sort_order,
    updated_at   = NOW();
```
